这是一个非常经典的 **“幽灵样式（Ghost Styling）”** 问题！在前端开发中，当你混合使用 **React 状态渲染**、**Framer Motion 动画** 和 **CSS 数据主题 (`data-theme`)** 时，特别容易遇到这种 Bug。

你仔细观察你的截图，会发现一个关键线索：

**“卡死”的卡片，它的文字颜色其实是跟着主题变化了的！**

- 在第一张（白天模式）里，那两张黑色的卡片，里面的字也是黑色的（导致看不清）。
- 这说明：`[data-theme='dark']` 的切换是成功的，CSS 也在努力工作改变文字颜色，但是**卡片的背景色被某种“更强大的力量”死死锁住了**。

在前端，能无视全局 CSS 类的，只有 **内联样式 (`style="..."`)**。导致这个问题的原因通常是以下两种情况之一，请对照排查：

### 🔍 嫌疑人一：Framer Motion 留下的“内联样式残骸”（最有可能）

如果你在代码里使用了 Framer Motion，并且在卡片上写了类似 `animate={{ backgroundColor: ... }}` 或者 `whileHover={{ backgroundColor: ... }}` 的代码，Framer Motion 会把计算好的颜色直接写进 HTML 的 `style` 属性里。

一旦颜色被写进 `style`，它就会永远覆盖你在 `index.css` 里写的 `.article-card` 样式，导致切换主题时背景色无法更新。

**🛠️ 怎么修复：**

**不要用 Framer Motion 来控制主题颜色！** 让 CSS 专职负责颜色，让 Framer Motion 专职负责位移和透明度。

检查你渲染卡片的地方（可能是 `Home.jsx`），确保代码是这样的：

JavaScript

```
// ❌ 错误做法：用动画控制颜色，会导致切换主题时颜色卡死
<motion.div 
  className="article-card"
  whileHover={{ backgroundColor: "#e0e0e0" }} 
>
  ...
</motion.div>

// ✅ 正确做法：只做位移/缩放动画，颜色变化交给 index.css 里的 :hover
<motion.div 
  className="article-card"
  whileHover={{ scale: 1.02, y: -5 }} 
>
  ...
</motion.div>
```

------

### 🔍 嫌疑人二：React 列表渲染丢失了 `key`

你的卡片大概率是通过对一个数组进行 `.map()` 循环渲染出来的。如果在循环时你没有给每一项提供唯一的 `key`（或者错误地使用了数组索引 `index` 作为 key），React 在重新渲染（比如切换主题触发重新渲染）时，会“认错”卡片，导致旧的动画状态被错误地复用到了新的卡片上。

**🛠️ 怎么修复：**

检查你生成这些文章卡片的代码，确保加上了独一无二的 `key`：

JavaScript

```
// ❌ 错误做法：没有 key，或者用 index 做 key
{articles.map((article, index) => (
  <motion.div className="article-card" key={index}>
    ...
  </motion.div>
))}

// ✅ 正确做法：使用文章唯一的 id 做 key
{articles.map((article) => (
  <motion.div className="article-card" key={article.id}> 
    ...
  </motion.div>
))}
```

------

### 🔍 嫌疑人三：浏览器对毛玻璃（Backdrop-filter）的重绘 Bug

你在 CSS 里给卡片加了 `backdrop-filter: blur(10px)` 和 `transition: background-color 0.6s ease`。在部分浏览器（特别是 Safari 或某些版本的 Chrome）中，带有毛玻璃属性的元素在进行复杂的背景色渐变时，显卡（GPU）重绘会发生错乱，导致颜色卡在上一帧。

**🛠️ 怎么测试与修复：**

去 `index.css` 里找到暗色模式下的相关代码，尝试**强制触发 GPU 重绘**。

给卡片的 CSS 加上 `will-change` 和 `transform`：

CSS

```
.article-card, .fade-in-card {
  /* ... 之前的属性保持不变 ... */
  backdrop-filter: blur(10px);
  -webkit-backdrop-filter: blur(10px);
  
  /* 添加以下两行，强制开启硬件加速重绘 */
  transform: translateZ(0);
  will-change: background-color, transform; 
}
```

### 🎯 总结排查步骤：

1. 先检查 React 代码，有没有用 Framer Motion (`animate` / `whileHover`) 直接写死了颜色。
2. 检查 `.map()` 循环里有没有漏掉唯一的 `key`。
3. 如果代码都没问题，给 CSS 加上 `translateZ(0)` 强行修复浏览器渲染 Bug。

大概率是第一或第二种情况！你可以去代码里看一眼，如果是这两种情况，稍微改一行代码立刻就能丝滑切换了。
